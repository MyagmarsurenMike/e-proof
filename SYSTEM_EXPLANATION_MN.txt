E-PROOF СИСТЕМИЙН АЖИЛЛАГААНЫ ТАЙЛБАР

Энэхүү баримт нь E-Proof системийн хэрхэн ажилладаг, ялангуяа файлууд болон өгөгдөл хэрхэн хадгалагддаг талаарх дэлгэрэнгүй тайлбар юм.

1. ЕРӨНХИЙ БҮТЭЦ
E-Proof нь Next.js фреймворк дээр суурилсан, блокчэйн технологи ашиглан баримт бичгийг баталгаажуулах систем юм.

2. ФАЙЛ ХАДГАЛАЛТ БА GIT (ЧУХАЛ)
Таны дурдсанчлан, хэрэглэгчийн оруулсан (upload хийсэн) файлуудыг Git рүү илгээхгүй байх нь маш чухал. Одоогийн тохиргоогоор энэ асуудлыг бүрэн шийдсэн байна:

- Файлын байршил: Бүх оруулсан файлууд төслийн `uploads/` хавтсанд хадгалагддаг.
- Git тохиргоо: `.gitignore` файлд `/uploads` гэж зааж өгсөн байгаа.
- Үр дүн: Та `git add .`, `git commit`, `git push` хийх үед `uploads` хавтас доторх файлууд хэзээ ч Git сервер (GitHub/GitLab) рүү хуулагдахгүй. Тэд зөвхөн таны локал компьютер эсвэл сервер дээр л үлдэнэ.

3. БАТАЛГААЖУУЛАЛТЫН ПРОЦЕСС
Систем дараах дарааллаар ажилладаг:

Алхам 1: Файл оруулах
Хэрэглэгч вебсайтаар дамжуулан файлаа оруулна (Drag & Drop эсвэл сонгох).

Алхам 2: Hash үүсгэх (Нууцлал)
Систем тухайн файлын агуулгаас "Cryptographic Hash" (дижитал хурууны хээ) үүсгэнэ.
- Жинхэнэ файл: `uploads/` хавтсанд хадгалагдана (эсвэл тохиргооноос хамаарч зөвхөн түр зуур боловсруулагдана).
- Hash код: Өгөгдлийн санд хадгалагдана.

Алхам 3: Блокчэйн рүү илгээх
Зөвхөн үүссэн Hash код л блокчэйн сүлжээ рүү илгээгдэнэ. Таны баримт бичгийн агуулга, нууц мэдээлэл хэзээ ч блокчэйн дээр ил гарахгүй.

Алхам 4: Баталгаажуулалт
Блокчэйн дээр гүйлгээ хийгдэж баталгаажсаны дараа, систем тухайн баримт бичгийг "VERIFIED" (Баталгаажсан) төлөвт шилжүүлнэ.

4. ӨГӨГДЛИЙН САН (DATABASE)
Систем нь PostgreSQL өгөгдлийн сан ашигладаг бөгөөд Prisma ORM-ээр дамжуулан харьцдаг.

Гол моделиуд (Schema.prisma дээрх):
- User: Хэрэглэгчийн бүртгэл, мэдээлэл.
- Document: Баримт бичгийн метадата (нэр, хэмжээ, төрөл), файлын hash, болон блокчэйн дээрх гүйлгээний мэдээлэл.
- VerificationStep: Баталгаажуулалтын алхам бүрийн түүх (Файл хуулсан -> Hash үүсгэсэн -> Блокчэйн рүү илгээсэн гэх мэт).
- File: Файлын физик байршил болон бусад мэдээллийг хадгалах модель.

5. ДҮГНЭЛТ
Энэхүү систем нь таны файлуудын нууцлалыг бүрэн хангахаар зохион бүтээгдсэн.
- Файлууд Git рүү орохгүй.
- Файлууд Блокчэйн рүү орохгүй (зөвхөн hash нь орно).
- Бүх үйлдэл өгөгдлийн санд бүртгэгдэж, хянах боломжтой.

6. АШИГЛАСАН ТЕХНОЛОГИУД
Энэхүү систем нь орчин үеийн, найдвартай дараах технологиудыг ашиглан бүтээгдсэн:

- **Frontend & Backend Framework**: Next.js 16 (React 19) - Вэбсайтын үндсэн бүтэц болон серверийн үйл ажиллагаа.
- **Програмчлалын хэл**: TypeScript - Кодын алдааг багасгаж, найдвартай байдлыг хангана.
- **Өгөгдлийн сан (Database)**: PostgreSQL - Хэрэглэгч болон баримт бичгийн мэдээллийг хадгалах.
- **ORM (Object-Relational Mapping)**: Prisma - Өгөгдлийн сантай харьцах гүүр.
- **Дизайн ба UI**: Ant Design & Tailwind CSS - Хэрэглэгчийн интерфэйс болон загварчлал.
- **Баталгаажуулалт (Auth)**: NextAuth.js - Хэрэглэгчийн нэвтрэх эрх, хамгаалалт.
- **Блокчэйн холболт**: Ethers.js & Web3.js - Блокчэйн сүлжээтэй харьцах, гүйлгээ хийх.
- **Нууцлал**: Bcryptjs & Jose - Нууц үг болон токенийг шифрлэх.
